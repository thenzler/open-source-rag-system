EDGE CASE TESTING:
- Ask about information not in documents
- Test with ambiguous questions
- Request impossible combinations
- Ask for opinions vs. facts
- Test with technical jargon

MULTILINGUAL TESTING:
- Ask questions in German and English
- Test technical terminology understanding
- Verify consistent responses across languages
- Test with mixed-language queries

PERFORMANCE TESTING:
- Ask rapid-fire questions
- Request complex multi-part answers
- Test with very long queries
- Verify response time consistency

TIPS FOR EFFECTIVE DEMONSTRATION:

1. START SIMPLE: Begin with basic factual questions
2. BUILD COMPLEXITY: Gradually increase question difficulty
3. SHOW VERSATILITY: Mix different document types
4. DEMONSTRATE ACCURACY: Verify specific details
5. TEST SYNTHESIS: Ask cross-document questions
6. SHOW LIMITATIONS: Test with unavailable information
7. HIGHLIGHT BENEFITS: Focus on practical applications
8. ENGAGE AUDIENCE: Ask follow-up questions
9. MEASURE PERFORMANCE: Track response quality
10. COLLECT FEEDBACK: Note user reactions

EXPECTED OUTCOMES:
- Accurate fact retrieval from multiple documents
- Intelligent synthesis of information
- Clear source attribution
- Appropriate handling of unavailable information
- Professional and helpful responses
- Consistent performance across question types
